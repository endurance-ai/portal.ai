## 2. Daydream 온보딩 (Style Passport)

### 수집하는 정보

| 단계 | 수집 항목 | 비고 |
|------|-----------|------|
| 1 | 이메일 / Apple ID | 계정 생성 |
| 2 | 이름, 생년월일 | 기본 프로필 |
| 3 | 평소 지출 범위 (예산) | 가격대 개인화 핵심 |
| 4 | 선호 브랜드 | 스크롤형 브랜드 디스커버리 UI |
| 5 | 사이즈 정보 | 핏 추천용 |

### 특징
- **의도적으로 가벼운 온보딩** — 5~6개 필드만 수집
- 체형 퀴즈 없음, 스타일 이미지 선택 없음, TPO 질문 없음
- **사용하면서 학습하는 전략** — 검색, 클릭, 저장 행동에서 신호 추출
- Style Passport는 "동적으로 진화하는 프로필" — 같은 검색어라도 유저마다 다른 결과

### 진화 방식
```
초기: 이름 + 생년월일 + 예산 + 브랜드 + 사이즈
  ↓ (사용)
학습: 검색 패턴, 클릭 이력, 저장 아이템, 컬렉션
  ↓ (축적)
성숙: 완전 개인화된 홈피드 + 검색 결과
```

> 참고: [PR Newswire — Daydream Launch](https://www.prnewswire.com/news-releases/daydream-launches-first-chat-based-shopping-agent-built-exclusively-for-fashion-302490581.html), [The New Consumer](https://newconsumer.com/2025/07/daydream-shopping-ai-chatgpt-google/)

## 3. Daydream "Say More" — 필터의 재발명

### 개념
전통적 필터(드롭다운, 체크박스, 슬라이더)를 **대화형 리파인먼트**로 대체.

### 사용 예시
| 유저 입력 | AI 해석 |
|-----------|---------|
| "Make it cheaper" | 가격대 하향 |
| "More colorful" | 채도 높은 아이템으로 교체 |
| "Different print or cut" | 패턴/실루엣 변경 |
| "Not too tight" | 핏을 루즈하게 조정 |
| "Opaque, not sheer" | 소재 필터링 |

### UX 설계
- 결과 화면 하단에 인터랙티브 프롬프트 영역
- **결과가 실시간 업데이트** — 초기 검색 상태로 돌아가지 않음
- 여러 리파인먼트를 **순차적으로 체이닝** 가능
- Red Antler (브랜딩 에이전시): "Say More는 브랜드 철학이자 제품 인터페이스"

> 참고: [Red Antler — Daydream Case Study](https://www.redantler.com/work/daydream), [Lizzy Helena Brown Review](https://lizzyhelenabrown.com/blogs/style/daydream-review)

## 5. 가격 필터 UX 패턴

### 5-1. 대화형 접근 (Daydream, Google AI Mode)

- 전통적 필터 UI 없음
- 자연어로 가격 조정: "under $100", "더 저렴하게"
- **장점**: 마찰 없음, 자연스러운 탐색
- **단점**: 정확한 범위 지정 어려움

### 5-2. 구조화 필터 (한국 앱 — 무신사, 지그재그, 에이블리)

| 앱 | 가격 필터 방식 | 특징 |
|----|---------------|------|
| **무신사** | 가격 범위 + 연령대 + 카테고리 | 성별이 최상위 탭 (무신사 멘 / 우신사) |
| **지그재그** | 단일 페이지에 모든 필터 | 온보딩: 나이 + 선호 스타일 + 쇼핑몰 선택 |
| **에이블리** | 단계별 필터 (좌→우 방향) | 카테고리별 다른 페이지에서 필터 |

### 5-3. 글로벌 럭셔리 (SSENSE, Net-a-Porter)

| 앱 | 가격 필터 방식 | 특징 |
|----|---------------|------|
| **SSENSE** | 듀얼 핸들 슬라이더 | 디자이너, 스타일, 가격, 카테고리, 사이즈, 컬러 |
| **Net-a-Porter** | 카테고리/디자이너/컬러/사이즈 (가격 약함) | 럭셔리 타겟이라 가격 필터 비중 낮음 |

### 5-4. 2025~2026 베스트 프랙티스

| 패턴 | 설명 | 효과 |
|------|------|------|
| **프리셋 칩** | "~5만", "5~10만", "10~20만", "20만~" | 1탭 선택, 가장 빠른 UX |
| **듀얼 핸들 슬라이더** | 최소/최대 핸들 | 시각적, 직관적 |
| **텍스트 입력 병행** | 슬라이더 옆 정확한 값 입력 | 접근성 |
| **히스토그램 오버레이** | 가격대별 상품 수 바 차트 | 빈 결과 방지 |
| **동적 카운트** | "142개 상품" — 슬라이더 이동 시 실시간 업데이트 | 즉각 피드백 |
| **필터 칩** | 결과 상단에 "가격: 5~10만 [x]" | 적용 상태 명확 |

> 참고: [BTNG.studio — Ecommerce Filter UX Patterns 2026](https://www.btng.studio/articles/top-ecommerce-ux-filter-design-patterns-practical-tips-for-2025/)

## 6. 성별 / 체형 입력 UX 트렌드

### 서비스별 접근 방식

| 서비스 | 방식 | 특징 |
|--------|------|------|
| **Daydream** | 안 물어봄 | 브랜드 선호 + 쿼리 맥락에서 추론 |
| **무신사** | 이진 탭 전환 | 무신사 멘(남성/네이비) vs 우신사(여성/그레이) |
| **에이블리/지그재그** | 여성 전용 플랫폼 | 앱 자체가 성별화 |
| **Stitch Fix** | 가입 시 선택 (Men/Women/Kids) | + 체형, 허리 높이, 노출 선호도 |
| **DressX** | 셀피 기반 | AI가 실제 체형 반영, 성별 질문 없음 |
| **Zalando** | 계정 + 구매이력 추론 | 명시적 질문 안 함 |

### 2025~2026 업계 트렌드
- **"All-gender" 카테고리 확산**: ZARA "Ungendered", ASOS "Fluid Fashion"
- **이진 분류에서 선호 기반으로 이동**: 성별 대신 체형 + 스타일 선호도
- **AI 추론 방식이 프로그레시브 방향**: MOODFIT의 `detectedGender` 접근이 오히려 트렌드에 부합
- **포괄적 사이징**: 성별화된 사이즈 차트 대신 체형 중심 추천

> 참고: [Standard Beagle — Gender-Neutral UX](https://standardbeagle.com/ux-design-for-gender-neutral-users/), [Glance AI — Inclusive Fashion 2026](https://glance.com/us/blogs/glanceai/fashion/ai-powering-inclusive-fashion)

## 7. MOODFIT 적용 제안

### 7-1. 즉시 추가 권장 (POC)

| 입력 항목 | 이유 | 구현 방식 |
|-----------|------|-----------|
| **가격대 선택** | 가격 혼재 문제 해결 (₩3만~₩300만) | 프리셋 칩: 💰/💰💰/💰💰💰 |
| **성별 확인/수정** | AI 추론 오류 대비 | AI 추론 → 유저 확인/수정 가능 |

### 7-2. MVP에서 추가 고려

| 입력 항목 | 이유 | 구현 방식 |
|-----------|------|-----------|
| **"Say More" 스타일 리파인먼트** | Daydream 킬러 기능 차용 | 결과 화면에서 "더 저렴하게", "더 캐주얼하게" 칩 |
| **선호 브랜드** | 검색 결과 개인화 | 분석 후 브랜드 선호도 학습 |

| 항목 | 이유 |
|------|------|

| 체형/셀피 입력 | Phase 3 이후 가능성, POC/MVP 범위 아님 |

## 8. Daydream vs MOODFIT 포지셔닝 매트릭스

```
                     텍스트 중심 ←────────────→ 이미지 중심
                          │                        │
         높은 온보딩     │  Stitch Fix            │
         투자            │                        │
                          │      Daydream          │
                          │  (챗 + Style Passport) │
                          │                        │
         ─────────────────┼────────────────────────┤
                          │                        │
         낮은 온보딩     │  Zalando Assistant      │  ★ MOODFIT
         투자            │  Google AI Mode         │  (이미지 = 전부)
                          │                        │
                          │           Pinterest     │
                          │           Lens          │
```

**MOODFIT의 위치**: 이미지 중심 + 낮은 진입 장벽

이것이 차별화이자 리스크 — 추가 입력 없이 정확한 추천이 가능한지가 핵심 검증 포인트.

## 9. 의사결정 필요 사항

> ⚠️ 아래 항목은 팀 내 결정이 필요합니다.

1. **가격대 칩을 업로드 전에 보여줄 것인가, 결과 후에 필터로 제공할 것인가?**
   - 업로드 전: 검색 정확도 ↑, 진입 마찰 ↑
   - 결과 후: 마찰 없음, 초기 결과에 가격 혼재

2. **"Say More" 스타일 리파인먼트를 MVP에 넣을 것인가?**
   - 넣으면: Daydream 대비 기능 격차 축소, 개발 비용 ↑
   - 안 넣으면: 룩 분해 + 핫스팟이라는 고유 차별점에 집중

3. **성별을 명시적으로 받을 것인가, AI 추론 유지할 것인가?**
   - 명시적: 검색 정확도 ↑, 글로벌 트렌드에 역행
   - AI 추론 + 수정 가능: 트렌드 부합, 오류 시 사용자가 교정
