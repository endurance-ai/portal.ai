# 빈티지 마켓플레이스 크롤링 조사

> 조사일: 2026-04-06
> 대상: 후루츠패밀리 (fruitsfamily.com) — 한국 최대 온라인 빈티지/세컨핸드 패션 마켓플레이스

## 1. 플랫폼 분석

| 항목 | 내용 |
|------|------|
| 플랫폼 유형 | Custom React SPA (CRA) + GraphQL 백엔드 |
| 인프라 | AWS (Elastic Beanstalk + CloudFront + API Gateway) |
| API | `web-server.production.fruitsfamily.com/graphql` (introspection 비활성화) |
| 렌더링 | SSR + 클라이언트 hydration (HTML 파싱 가능) |
| 결제 | iamport, TossPayments, NicePay |

**Cafe24/Shopify 아님** — 기존 크롤러 엔진 사용 불가, 별도 엔진 필요.

### 규모

| 항목 | 수치 |
|------|------|
| 상품 수 | ~100,000개 (sitemap 기준, 100페이지 × 1,000개) |
| 카테고리 | 13개 (상의, 아우터, 하의, 신발, 가방, 모자, 액세서리, 주얼리, 치마, 원피스, 향수, 라이프, 굿즈) |
| 브랜드 | ~12,876개 |
| 셀러 | ~27,000명 |
| 컬렉션 | 383개 |

### URL 패턴

- 상품: `/product/{shortId}/{slug}`
- 검색: `/search/{keyword}?sort=POPULAR`
- 컬렉션: `/collection/{shortId}/{slug}`
- 브랜드: `/brand/{BrandName}`

## 2. 크롤링 방식 비교

| 방식 | 속도 (10만개) | 법적 리스크 | 비고 |
|------|-------------|-----------|------|
| GraphQL API 직접 호출 | 20~40분 | **높음** (역설계 금지 위반) | 비권장 |
| Sitemap + HTML fetch | 3~5시간 | 중간 | **권장** — SSR 확인됨, 브라우저 불필요 |
| Playwright SPA 렌더링 | 7~9시간 | 중간 | 차선책 |

선별 수집 (1~2만개) 시 Sitemap + HTML 방식으로 **1~2시간** 예상.

## 3. 법적 리스크 분석

### 관련 법률

| 법률 | 리스크 | 핵심 조항 |
|------|--------|----------|
| 부정경쟁방지법 (카목) | **높음** | "상당한 투자로 만든 성과의 무단 이용" |
| 저작권법 (DB제작자 권리) | **높음** | 체계적/반복적 수집 → "상당 부분 복제" 인정 가능 |
| 정보통신망법 제48조 | 중간 | 기술적 보호조치 없으면 형사 무죄 가능 (대법원 2021도1533) |
| 개인정보보호법 | 중간~높음 | C2C 마켓 → 셀러 개인정보 포함 가능 |
| 이미지 저작권 | **높음** | 셀러 착용샷/스타일링 = 창작물 인정 가능성 높음 |

### 주요 판례

- **여기어때 vs 야놀자 (대법원 2021도1533)**: 형사 무죄, but 민사에서 부정경쟁행위 손해배상 인정
- **잡코리아 vs 사람인**: IP 차단 우회 → 형사 유죄 + DB제작자 권리 침해
- 형사는 피할 수 있어도 **민사 소송 리스크는 항상 존재**

### 후루츠패밀리 구체적 상황

- **robots.txt**: 모든 경로 허용 (단, 법적 면책은 아님)
- **이용약관**: "무단 데이터 수집", "역설계" **명시적 금지**
- **C2C 특수성**: 데이터 소유권이 플랫폼 + 개인 셀러에 분산 → 이중 리스크

### 리스크 완화 방안

| 수준 | 방안 |
|------|------|
| **최소 리스크** | 공식 제휴/API 협약 체결 (가장 안전) |
| 중간 리스크 | 사실 정보만 최소 수집 + 이미지 링크만 + 셀러 정보 제외 + 출처 명시 + 서버 부하 최소화 |
| **절대 금지** | GraphQL 리버스엔지니어링, 이미지 직접 저장, 전체 DB 미러링, IP 차단 우회 |

## 4. 결론: 현 단계에서는 보류

**빈티지 마켓 특성상 크롤링 실효성이 낮음:**
- 1점 한정 상품 → 빠른 sold out → 크롤링 데이터가 금방 stale
- 실시간 재고 동기화 없이는 사용자 경험 저하
- 법적 리스크 대비 ROI가 낮음

**권장 방향:**
1. 현재는 **Cafe24 기반 편집샵/브랜드몰 확장**에 집중 (시즌 단위 안정 재고)
2. 후루츠패밀리는 **공식 제휴 + API 연동**으로 실시간 재고 반영이 가능해질 때 도입
3. 제휴 시 어필리에이트/수익 공유 모델로 상호 이익 구조 설계
