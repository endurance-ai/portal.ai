# Fashion Genome DB 연동 로드맵

> - **작성일**: 2026-03-31
> - **최종 수정**: 2026-04-02
> - **대상**: 기획자 공유용
> - **프로젝트**: portal.ai — AI 이미지 기반 패션 무드 분석 & 상품 추천

---

## 배경

portal.ai는 유저가 패션 이미지를 올리면 AI가 무드를 분석하고 유사 상품을 추천하는 서비스입니다.

동료가 구축한 **Fashion Genome DB**와 연동하면, 단순 키워드 검색 대신 **브랜드 감도 기반 추천**이 가능해집니다.

| 항목 | 내용 |
|------|------|
| Fashion Genome DB 규모 | 1,079개 브랜드 |
| 스타일 분류 체계 | 15개 스타일 노드 (A-1 ~ K) |
| 감도 태그 | 12종 (미니멀, 하이엔드, 스트리트 등) |
| 크롤링 플랫폼 | 9개 편집샵 (상품 수 추정 8,000~10,000+) |

## 팀원에게 요청할 DB 작업

> 아래 표에서 **"왜 필요한가"** 부분이 가장 중요합니다.
> 개발팀이 코드로 처리할 수 없고, DB 원본에서만 해결 가능한 것들만 정리했습니다.

### 지금 바로 — POC에 직접 영향

#### 1. source_platforms가 비어있는 의류 브랜드 27개

**현재 상태**: 27개 브랜드에 `source_platforms`(이 브랜드를 어디서 살 수 있는지)가 비어있습니다.

**왜 필요한가**: 유저가 사진을 올리면, AI가 "이 사진은 C노드(미니멀 컨템퍼러리)에 해당"이라고 분석합니다. 그러면 C노드에 속한 브랜드 149개 중에서 상품을 찾습니다. 그런데 그 149개 브랜드 중 일부가 "어디서 파는지 모르는" 상태면, 해당 브랜드 상품을 크롤링할 수 없고 추천에도 나타나지 않습니다.

**구체적으로 비어있는 브랜드 예시**:

| 브랜드 | 노드 | 카테고리 |
|--------|------|---------|
| 그레일즈 | A-2 하이엔드 럭셔리스트릿 | 의류 |
| 로우클래식 | F 미니멀 페미닌 | 의류 |
| 마르니 | A-2 하이엔드 럭셔리스트릿 | 의류 |
| 아워레가시 | D 컨템퍼러리 캐주얼 | 의류 |
| 아크테릭스 | G 테크니컬 고프코어 | 의류 |
| 슈프림 | H 스트릿 캐주얼 | 의류 |
| 스투시 | H 스트릿 캐주얼 | 의류 |
| ... 외 20개 | | |

**요청**: 각 브랜드가 우리 크롤링 대상 편집샵(아래 9개)에 입점되어 있는지 확인하고, 있으면 해당 플랫폼명을 채워주세요.

> 크롤링 대상 편집샵: 샵아모멘토, 슬로우스테디클럽, 아데쿠베, 이티씨서울, VISUAL AID, 아이엠샵, 8디비전, 스컬프스토어, 프레이트

만약 우리 크롤링 대상에 없는 브랜드라면, 비워두셔도 괜찮습니다 (향후 플랫폼 추가 시 다시 확인).

---

#### 2. "보류" 상태 아이웨어 브랜드 13개 — 방향 결정 필요

**현재 상태**: 아래 13개 브랜드가 `final_node_name = "보류"` 상태입니다. 전부 SSENSE 출처의 아이웨어(선글라스/안경) 브랜드입니다.

| 브랜드 | 카테고리 |
|--------|---------|
| AKILA, Akoni, Cartier, Cutler and Gross, Dita, DMY Studios, Le Specs, Oliver Peoples, Port Tanger, Prada Eyewear, Ray-Ban, RETROSUPERFUTURE, Velvet Canyon | 아이웨어 |

**왜 결정이 필요한가**: "보류" 브랜드는 현재 서비스에서 완전히 무시됩니다. 유저가 선글라스가 포함된 사진을 올려도, 이 브랜드들은 추천에 나타나지 않습니다.

**선택지**:
- **(A) 노드 배정**: 각 아이웨어 브랜드의 감도에 맞는 스타일 노드를 배정. 예) Ray-Ban → H 스트릿 캐주얼, Oliver Peoples → C 미니멀 컨템퍼러리
- **(B) "제외" 처리**: 아이웨어는 우리 서비스 범위 밖으로 판단하고 `category_type = "제외"`로 변경
- **(C) 유지 (나중에)**: 지금은 보류 유지, MVP에서 액세서리 추천 기능 추가 시 재검토

→ 현재 SSENSE를 크롤링하지 않으므로, **(B) 또는 (C)** 가 현실적입니다.

### 다음 단계 — 추천 품질 향상

#### 3. 저가대(~10만원) 브랜드가 거의 없음

**현재 상태**: price_band 분포를 보면 고가 편향이 심합니다.

| 가격대 | 브랜드 수 | 비율 |
|--------|----------|------|
| 30~50만 | 517 | **48%** |
| 50~100만 | 308 | 29% |
| 10~30만 | 233 | 22% |
| 100만+ | 12 | 1% |
| **~10만** | **2** | **0.2%** |

**왜 문제인가**: MVP에서 가격대 필터를 넣으면 유저가 "5만원 이하" 또는 "5~15만원"을 선택할 수 있습니다. 그런데 10만원 이하 브랜드가 2개뿐이면, 저가대를 선택한 유저에게 추천할 브랜드가 거의 없습니다. 실제로 편집샵 크롤링 상품 중에는 10만원 이하 상품이 꽤 있는데, 해당 브랜드들이 DB에 "저가대"로 분류되어 있지 않습니다.

**확인 필요**: 이것이 의도적인 건지 (편집샵 큐레이션 서비스라 저가 브랜드는 타겟이 아님), 아니면 채워야 하는 건지 논의가 필요합니다.

→ 의도적 제외라면 그대로 두되, MVP 가격 필터 UI에서 "~10만" 옵션을 빼는 것을 고려합니다.

#### 4. brand_keywords / 인접 노드 검증

| 항목 | 현재 상태 | 요청 | 이유 |
|------|-----------|------|------|
| **brand_keywords** | 1,079개 전체 입력됨, 표준 어휘 100% 준수 | 노드별 샘플 50개 스팟체크 | 태그/키워드 표준화는 완벽하지만, 개별 브랜드에 **맞는 키워드**가 배정되었는지는 별개 문제. 예) "드레이프" 키워드가 실제로 드레이프 실루엣이 핵심인 브랜드에만 붙어있는지 |
| **인접 노드 관계** | 코드에 정의됨 (예: C의 인접 = D, B-2, E, F-3) | 인접 관계가 실제 감도와 맞는지 검토 | AI가 "이 사진은 C노드 85%, D노드 60%"라고 분석하면, C노드 브랜드를 먼저 보여주고, 부족하면 D노드까지 확장합니다. 이 "C→D 확장"이 맞으려면 인접 관계가 정확해야 합니다 |

### 3단계: 운영

> 목표: **발견의 재미 + 개인화**

| 항목 | 작업 |
|------|------|
| 시즌 컬렉션 태깅 | 시즌별 추천 |
| 브랜드 인접 관계 매핑 | "이 브랜드를 좋아하면 이것도" |
| 노드 정확도 모니터링 | 사용자 피드백 → AI 분류 검증 |
| 선호 브랜드 학습 | 온보딩 또는 사용 중 자동 축적 |

## 매칭 파이프라인 (전체 흐름)

```
[유저] 이미지 업로드 + 성별 확인
  │
  ▼
[AI 분석] GPT-4o-mini Vision
  ├─ 스타일 노드     예) C. 미니멀 컨템퍼러리 (85%)
  ├─ 감도 태그       예) 미니멀, 하이엔드
  ├─ 브랜드 키워드    예) 드레이프, 모노톤, 구조적
  └─ 아이템별 키워드   예) 코트: oversized grey wool coat
  │
  ▼
[매칭] Fashion Genome DB
  ├─ 1차: primary 노드 브랜드 풀
  │      + 감도 태그 부스트 (AI 태그 ↔ 브랜드 태그 겹치면 가산점)
  │      + 제외 브랜드 필터 (향/라이프스타일 자동 제거)
  ├─ 2차: secondary/인접 노드 확장
  ├─ 필터: 성별, category_type ≠ 제외, (MVP) price_band
  └─ 3차: 전체 검색 fallback (키워드만)
  │
  ▼
[상품] 자체 DB 우선, SerpApi fallback
  └─ 아이템당 상위 5개 (브랜드 다양성 보장: max 2/brand)
```
