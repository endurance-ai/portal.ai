### 시장에서 발견한 기회

1. 대부분 **자체 커머스 플랫폼 안에서만** 추천 → 독립형 서비스가 거의 없음
2. 국내에 "이미지 → 스타일 분석 → 크로스플랫폼 추천" B2C 서비스는 **빈자리**
3. 해외 서비스(Daydream 등)는 한국/아시아 브랜드 커버리지가 약함
4. 2026년 AI 패션 시장은 "실험 → 인프라"로 전환 중 (Vogue 보도)

> 한 줄 요약: **지그재그/에이블리 = "우리 매장 안에서 추천"**, 우리 = **"어디서든 찾아주는 AI 스타일리스트"**

## 3. 프로토타입 시연 — Daydream과의 차별점

> 현재 구현된 프로토타입으로 시연합니다.

### 시연 흐름

```
이미지 업로드 (드래그 & 드롭)
    ↓
AI 스캔 애니메이션 (분석 중 화면)
    ↓
결과 화면:
  - 원본 이미지 위 핫스팟 표시
  - 각 아이템(상의/하의/아우터/신발/악세서리) 분해
  - 커넥터 라인으로 아이템 → 상품 카드 연결
  - 무드 태그 + 컬러 팔레트 + 스타일 요약
```

### Daydream vs MOODFIT — 핵심 차별점 5가지

#### A. Look Decomposition — 룩 분해 비주얼라이제이션

Daydream은 이미지를 넣으면 **비슷한 상품 목록을 나열**한다.
우리는 이미지를 넣으면 **룩 전체를 분해**해서 각 아이템에 핫스팟 + 커넥터 라인으로 연결한다.

→ 패션 매거진 에디토리얼 같은 **"인터랙티브 룩북"** 경험.
→ 시각적 임팩트가 커서 **SNS 바이럴 가능성**이 높음.

#### B. 크로스 가격대 매칭

Daydream은 럭셔리 중심 (Net-A-Porter, SSENSE).
우리는 **같은 스타일을 ₩30,000 ~ ₩3,000,000까지** 가격대별로 보여준다.

```
"이 THE ROW 코트 스타일을 UNIQLO 가격에서도 찾아줄게"
```

→ 가격 민감한 유저 (학생, 20대) 공략. Daydream과 타겟이 자연스럽게 분리됨.

#### C. Mood DNA — 스타일 분석 리포트

단순 상품 추천이 아닌 **무드 분석 데이터** 제공:
- 무드 태그 + confidence score
- 추출 컬러 팔레트 (oklch 기반)
- 에디토리얼 톤 스타일 요약 (vibe, season, occasion)

→ 유저가 **자기 스타일을 "이해"**하게 되는 경험. "내 스타일은 이거였어!" → SNS 공유 유도.

#### D. 웹 퍼스트

Daydream은 iOS 앱 전용 → 앱스토어 설치 필요.
우리는 **웹 우선** → URL 하나로 접근 가능.

→ "이 사이트에 사진 넣어봐" 한 마디로 공유 가능. 바이럴에 압도적으로 유리.

#### E. 아시아 브랜드 커버리지

Daydream은 서양 럭셔리 중심.
우리는 **무신사, W컨셉, 29CM, ZARA, COS** 등 한국/아시아 브랜드 포함.

→ 한국 시장 + 해외 K-fashion 관심 유저 동시 공략.

### 포지셔닝 비교

```
Daydream = "AI 패션 챗봇" → 대화로 쇼핑  → "무엇을 살까?"에 답한다
MOODFIT  = "AI 룩 분석기" → 사진으로 해부 → "이 스타일이 뭔데? 어디서 살 수 있어?"에 답한다
```

## 4. 기술 파이프라인 & 구현 포인트

### 전체 파이프라인

```
사용자가 옷 사진 업로드
        ↓
[1단계] 이미지 분석 — GPT-4o-mini Vision
        → 스타일, 색감, 소재, 핏, 분위기 추출
        → 아이템별 분해 (상의/하의/아우터/신발/악세서리)
        → 무드 태그 + confidence score
        ↓
[2단계] 상품 검색 — SerpApi (Google Shopping)
        → 아이템별 검색 쿼리 생성
        → 성별 자동 감지 → 검색 필터 반영
        → 관련성 + 평점 스코어링 → 상위 4개 선별
        ↓
[3단계] 결과 UI — 인터랙티브 룩 분해 화면
        → 원본 이미지 위 핫스팟
        → 커넥터 라인 → 상품 카드 연결
        → 무드 리포트 + 컬러 팔레트
```

### 현재 구현 상태 (POC)

| 구성요소 | 현재 | 비고 |
|---------|------|------|
| 프레임워크 | Next.js 16 (App Router) + Vercel | Turbopack, pnpm |
| UI | React 19 + Tailwind 4 + shadcn/ui + framer-motion | |
| 이미지 분석 | **GPT-4o-mini Vision** | 건당 ~$0.003 (약 4원) |
| 상품 검색 | **SerpApi** (Google Shopping) | 월 100회 무료 |
| 벡터 DB | 미구현 | Qdrant 예정 |
| 브랜드 DB | 미구현 | 향후 핵심 자산 |

### 향후 구현 필요 사항

| 항목 | 설명 | 우선순위 |
|------|------|---------|
| 벡터 DB (Qdrant) | 브랜드/스타일 벡터 검색 | 높음 |
| 브랜드 DB 구축 | 50개 브랜드 수동 등록 → 임베딩 | 높음 |
| 어필리에이트 API | 쿠팡 파트너스, 무신사 등 연동 | 중간 |
| 임베딩 모델 | text-embedding-3-small | 높음 |
| Mood DNA 공유 | SNS 공유용 카드 이미지 생성 | 중간 |

## 8. 로드맵

### Phase 1: POC (현재 ~ 4주)

```
✅ 이미지 → 룩 분해 + 무드 분석 (구현 완료)
✅ SerpApi 상품 검색 연동 (구현 완료)
✅ 3-screen UI: Upload → Analyzing → Result (구현 완료)
⬜ Mood DNA 리포트 공유 기능 (SNS 바이럴)
⬜ 10~20명 대상 테스트 → 피드백 수집
⬜ 핵심 지표 측정: 추천 정확도, 재사용률, NPS
```

### Phase 2: MVP

```
⬜ 브랜드 DB 구축 (50개 브랜드 + Qdrant 벡터 DB)
⬜ 크로스플랫폼 상품 검색 (어필리에이트 API 연동)
⬜ 가격대 필터 (₩ / ₩₩ / ₩₩₩)
⬜ 유저 히스토리 → 스타일 프로필 (Style Passport 대응)
⬜ "이 스타일 더 보기" → 연관 룩 추천
```

### Phase 3: Growth

```
⬜ 모바일 앱 (PWA 또는 네이티브)
⬜ 브랜드 파트너십 → 공식 API 연동 (실시간 재고/가격)
⬜ 커뮤니티 — 유저가 올린 룩 갤러리
⬜ 다국어 지원 (EN, KO, JP, ZH)
⬜ 운영 자동화: 유저 피드백 → AI 분류 → JIRA 자동 생성
```

### 판단 기준

```
POC 검증 후:
  → 긍정적 (추천 유용, 재사용 높음) → 베타 런칭 (100명 대상)
```
