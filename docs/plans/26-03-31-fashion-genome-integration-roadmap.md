# Fashion Genome DB 연동 로드맵

> - **작성일**: 2026-03-31
> - **대상**: 기획자 공유용
> - **프로젝트**: MOODFIT — AI 이미지 기반 패션 무드 분석 & 상품 추천

---

## 배경

MOODFIT은 유저가 패션 이미지를 올리면 AI가 무드를 분석하고 유사 상품을 추천하는 서비스입니다.

동료가 구축한 **Fashion Genome DB**와 연동하면, 단순 키워드 검색 대신 **브랜드 감도 기반 추천**이 가능해집니다.

| 항목 | 내용 |
|------|------|
| Fashion Genome DB 규모 | 1,079개 브랜드 |
| 스타일 분류 체계 | 15개 스타일 노드 (A-1 ~ K) |
| 감도 태그 | 12종 (미니멀, 하이엔드, 스트리트 등) |

### 기존 문제

- AI가 자유형 영어 키워드만 추출 → DB의 스타일 노드와 매칭 불가
- 예: AI가 "minimalist clean silhouette"라고 출력해도 DB의 "C. 미니멀 컨템퍼러리" 노드와 연결할 방법이 없었음

### 해결 (이번 세션)

- AI 프롬프트에 15개 노드 체계를 주입하여 **구조화된 출력** 추가
- `fashion-genome.ts`로 노드/태그를 코드 상수(ENUM)화 완료
- 이제 AI가 "이 이미지는 노드 C (미니멀 컨템퍼러리), 감도: 미니멀+하이엔드"처럼 출력

## 스테이지별 로드맵

### 1단계: POC (지금~)

> 목표: **노드 매칭으로 브랜드 풀 생성이 되는지 검증**

#### DB 보완 사항

| 항목 | 현재 상태 | 필요 작업 | 이유 |
|------|-----------|-----------|------|
| AI 프롬프트 | ✅ 완료 | 15개 노드 체계 주입 완료 | — |
| fashion-genome.ts | ✅ 완료 | 노드/태그 ENUM화 완료 | — |
| sensitivity_tags | 자유형 콤마 구분 텍스트 | 12종 기준으로 일관성 점검 | 브랜드마다 표기가 다를 수 있음 (예: "미니멀" vs "미니말") |
| ssense_gender_scope | SSENSE 외 브랜드는 비어있음 | 최소 men / women / unisex 중 하나 채우기 | 성별 필터 없으면 남성 사진에 여성 브랜드가 추천됨 |

**핵심 포인트**: 노드 매칭만으로도 "이 사진과 비슷한 감도의 브랜드 풀"을 만들 수 있습니다. 단, 성별 필터가 없으면 결과가 엉뚱해집니다.

#### 유저에게 받을 정보

| 정보 | 방식 | 근거 |
|------|------|------|
| 성별 확인 | AI 추론값을 보여주고, 유저가 수정 가능 | AI 성별 추론 정확도 ~85%. 오류 시 검색 결과 전체가 틀어짐. Daydream/Zalando도 이 방식 사용 |

**UI 제안**: 분석 결과 화면에서 **Men / Women / All** 탭. 기본값은 AI가 추론한 성별.

---

### 2단계: MVP

> 목표: **실제 쓸 만한 추천 경험 만들기 (가격 + 목적 필터)**

#### DB 보완 사항

| 항목 | 현재 상태 | 필요 작업 | 이유 |
|------|-----------|-----------|------|
| price_band | 18 / 1,079개만 있음 | 전체 브랜드 가격대 채우기 | 가격 필터링의 핵심. 없으면 3만원~300만원이 섞여 나옴 |
| 노드별 대표 키워드 사전 | 없음 | 노드별 영어 키워드 목록 추가 | AI가 뽑은 키워드와 노드를 2차 검증할 때 사용 |
| 아이템 카테고리별 브랜드 강점 | 없음 | 예: "이 브랜드는 아우터가 강함" 태깅 | "코트는 A 브랜드, 바지는 B 브랜드"식 추천 가능 |

**핵심 포인트**: 가격대 없이는 3만원짜리와 300만원짜리가 같이 나와서 사용자 신뢰도가 떨어집니다. 아이템별 강점 태깅이 있으면 각 아이템에 맞는 브랜드를 따로 추천할 수 있습니다.

#### 유저에게 받을 정보

| 정보 | 방식 | 근거 |
|------|------|------|
| 가격대 | 프리셋 칩: ~5만 / 5~15만 / 15~30만 / 30만+ | 모든 경쟁 서비스가 가격 필터 제공. 없으면 UX 파괴 |
| 쇼핑 목적 | 선택 칩 (optional): Daily / Office / Date / Travel | 같은 미니멀이라도 출근용 vs 데이트용으로 브랜드 감도가 다름 |
| "Say More" 리파인먼트 | 결과 하단 대화형 칩: "더 캐주얼하게", "더 다크하게" 등 | Daydream의 킬러 기능. 결과를 한 번에 맞추기 어려우므로 조정 수단 필요 |

**UI 제안**:
- **가격대**: 업로드 전 또는 결과 화면 상단에 1-tap 칩
- **쇼핑 목적**: 선택 칩 (선택 안 해도 됨)
- **Say More**: 결과 화면 하단 인터랙티브 칩

### 3단계: 운영

> 목표: **발견의 재미 + 개인화**

#### DB 보완 사항

| 항목 | 필요 작업 | 이유 |
|------|-----------|------|
| 시즌 컬렉션 태깅 | 브랜드별 S/S, F/W 등 시즌 정보 | 시즌에 맞는 추천 |
| 브랜드 인접 관계 | 브랜드 간 "비슷한 감도" 매핑 | "이 브랜드를 좋아하면 이것도" 추천 |
| 노드 정확도 모니터링 | 사용자 피드백 기반으로 AI 분류 검증 | AI가 붙인 노드가 실제와 맞는지 지속 확인 |

**핵심 포인트**: 운영 단계에서는 정확도보다 "발견의 재미"와 개인화가 중요합니다. 사용자 행동 데이터를 노드 체계에 피드백합니다.

#### 유저에게 받을 정보

| 정보 | 방식 | 근거 |
|------|------|------|
| 선호 브랜드 | 온보딩 또는 사용 중 학습 | 1,079개 브랜드 중 선호 브랜드로 노드를 역추론 가능 |
| 사이즈 | 구매 전환 단계에서 입력 | 실제 구매로 이어지려면 사이즈 정보 필요 |
| 스타일 히스토리 | 이전 분석 결과 기반 자동 축적 | 반복 사용할수록 개인화 정확도 향상 |

## 매칭 파이프라인 (전체 흐름)

아래는 유저 이미지가 브랜드 추천으로 이어지는 전체 과정입니다.

```
[유저] 이미지 업로드 + 성별 확인
  │
  ▼
[AI 분석] 이미지에서 추출하는 것들:
  ├─ 스타일 노드  예) C. 미니멀 컨템퍼러리
  ├─ 감도 태그    예) 미니멀, 하이엔드
  └─ 아이템별 키워드  예) 코트: 오버사이즈, 울, 블랙
  │
  ▼
[매칭] Fashion Genome DB에서 필터링:
  ├─ 노드 C에 해당하는 브랜드 풀 (예: 149개)
  ├─ 성별 필터 적용 (POC)
  └─ 가격 필터 적용 (MVP)
  │
  ▼
[결과] 유저에게 보여주는 것:
  ├─ 추천 브랜드 리스트
  └─ 아이템별 유사 상품 (SerpApi 검색)
```

## 스테이지별 요약

| 구분 | POC | MVP | 운영 |
|------|-----|-----|------|
| **DB 작업** | 성별 채우기, 감도 태그 정리 | 가격대 채우기, 키워드 사전, 아이템 강점 | 시즌, 인접 관계, 정확도 모니터링 |
| **유저 입력** | 성별 확인 | 가격대, 목적, Say More | 선호 브랜드, 사이즈, 히스토리 |
| **추천 방식** | 노드 매칭 → 브랜드 풀 | + 가격/목적 필터 | + 개인화 + 발견 추천 |
| **기대 효과** | "비슷한 감도의 브랜드"가 나오는지 확인 | 실제로 쓸 만한 추천 | 반복 사용 + 구매 전환 |
