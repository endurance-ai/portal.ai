## 2. 서비스 파이프라인 & 필요 AI 모델

### 전체 흐름

```
사용자가 옷 사진 업로드
        ↓
[1단계] 이미지 분석 (멀티모달 LLM)
        → 스타일, 색감, 소재, 핏, 분위기 추출
        → "캐주얼, 오버핏, 어두운 톤, 데님 소재, 스트릿 무드"
        ↓
[2단계] 스타일 벡터화 (임베딩 모델)
        → 추출된 스타일 텍스트를 벡터로 변환
        ↓
[3단계] 브랜드 DB 검색 (벡터 DB + RAG)
        → 사전에 구축한 브랜드/상품 DB에서 유사 스타일 검색
        → "이 스타일과 매칭되는 브랜드: A, B, C"
        ↓
[4단계] 추천 답변 생성 (LLM)
        → 브랜드 정보 + 스타일 분석 결과 → 자연어 추천문 생성
        → "이 스타일은 무신사 스탠다드의 오버핏 데님 라인과 잘 어울립니다..."
        ↓
사용자에게 추천 결과 + 상품 링크 제공
```

### 단계별 모델 선택

| 단계 | 역할 | 추천 모델 (API) | 대안 (자체호스팅) | 비용/건 |
|------|------|----------------|-----------------|---------|
| **1. 이미지 분석** | 옷 사진 → 스타일 키워드 추출 | **GPT-4o-mini** (비전) | Qwen3.5-4B (멀티모달) | ~$0.003 |
| **2. 임베딩** | 텍스트 → 벡터 변환 | **text-embedding-3-small** | BGE-M3 (자체호스팅) | ~$0.0001 |
| **3. 벡터 검색** | 유사 브랜드 검색 | Qdrant / Pinecone | Qdrant (자체호스팅) | 무료~$0.001 |
| **4. 답변 생성** | 추천문 생성 | **GPT-4o-mini** | Qwen3.5-9B | ~$0.001 |

### 대안 구성 (품질 우선)

| 단계 | 모델 | 비용/건 | 언제 쓰나 |
|------|------|---------|----------|
| 1. 이미지 분석 | **Claude Sonnet** (비전) | ~$0.01 | 복잡한 스타일 (패턴, 브랜드 로고 인식) |
| 4. 답변 생성 | **Claude Sonnet** | ~$0.005 | 한국어 품질이 중요할 때 |

## 5. 브랜드 DB 구축 (핵심 자산)

서비스의 진짜 가치는 AI 모델이 아니라 **브랜드 DB의 품질**입니다.

### 필요한 데이터

```
브랜드별:
  - 브랜드명, 가격대 (저가/중가/고가)
  - 대표 스타일 키워드 (캐주얼, 포멀, 스트릿, 미니멀 등)
  - 타겟 연령/성별
  - 대표 상품 이미지 + 설명
  - 공식 쇼핑몰 URL

상품별:
  - 상품명, 가격
  - 카테고리 (상의/하의/아우터/신발 등)
  - 스타일 태그
  - 상품 이미지
  - 구매 링크
```

### 데이터 수집 방법

| 방법 | 난이도 | 비용 | 데이터 질 |
|------|--------|------|----------|
| **무신사/W컨셉 등 크롤링** | 중 | 낮음 | 중 (법적 리스크 있음) |
| **브랜드 공식 API 연동** | 높 | 없음 | 높 (파트너십 필요) |
| **어필리에이트 플랫폼** | 낮 | 없음 | 중 (CJ, 쿠팡 파트너스 등) |
| **수동 큐레이션** | 높 | 인건비 | 최고 |

### 추천: 초기 MVP

```
1. 무신사 스탠다드, 지그재그, W컨셉 등 주요 플랫폼 50개 브랜드 수동 등록
2. 브랜드당 대표 스타일 키워드 + 가격대 + 대표 이미지 5~10장
3. 이 데이터를 벡터 DB에 임베딩
4. 사용자 이미지 → 스타일 추출 → 벡터 유사도 검색 → 브랜드 매칭
```

## 7. 결론 및 권장사항

### 시장 기회

```
✅ 국내에 "독립형 이미지 → 브랜드 추천" B2C 서비스는 거의 없음
✅ 해외에는 Daydream, DressX 등이 있지만 한국 브랜드 커버리지 부족
✅ 기술적으로 GPT-4o-mini 비전이면 충분히 구현 가능
✅ POC 비용이 매우 낮음 (월 1~10만원)
```

### 리스크

```
⚠️ 브랜드 DB 구축/유지가 핵심이자 가장 큰 노력
⚠️ 무신사/지그재그 같은 대형 플랫폼이 동일 기능을 자체 구현할 수 있음
⚠️ 수익화까지 유료 구독자 70명+ 필요
⚠️ 패션은 트렌드가 빨리 바뀌어서 DB 지속 업데이트 필요
```

### 권장 다음 단계

```
1. POC (4주, 비용 ~10만원):
   → 50개 브랜드 DB + 이미지 분석 파이프라인 구축
   → 주변 10~20명에게 테스트

2. 검증 (2주):
   → "이 추천이 실제로 유용한가?" 피드백 수집
   → 핵심 지표: 추천 정확도, 재사용률, NPS

3. 판단:
   → 긍정적 → 베타 런칭 (100명 대상)
   → 부정적 → 피벗 또는 중단 (손실 최소)
```

## 8. 운영 자동화: QA/유저 피드백 → JIRA 자동화

> 팀 구성: 2명 (개발 1 + 사업기획 1)
> 목표: 유저 피드백 수집 → AI 분석 → JIRA 태스크 자동 생성

### 전체 플로우

```
[유저]                    [수집]              [AI 분석]           [태스크 관리]
  │                        │                    │                    │
  ├─ 앱 내 피드백 ────→ 오픈클로(Cello) ──→ Claude ──────→ JIRA 자동 생성
  ├─ 앱스토어 리뷰 ──→ 오픈클로 수집    ──→ 분류/우선순위 ──→ Slack 알림
  ├─ CS 문의 ────────→ 오픈클로 수집    ──→ 버그/기능요청 분류 → 담당자 배정
  └─ SNS/커뮤니티 ──→ 오픈클로 수집    ──→ 감성 분석      → 주간 리포트
```

### 단계별 처리

```
[1] 수집 (오픈클로)
    → 인앱 피드백 위젯
    → 앱스토어 리뷰 자동 수집
    → CS 채널 통합

[2] AI 분석 (Claude API / Claude Code MCP)
    → 피드백 분류: [버그 / 기능요청 / UX개선 / 칭찬 / 기타]
    → 긴급도 판단: [긴급 / 보통 / 낮음]
    → 중복 감지: 기존 JIRA 이슈와 유사도 비교
    → 요약: 피드백 원문 → 1줄 요약 + 액션 아이템

[3] JIRA 자동 생성
    → 분류 결과에 따라:
      버그     → JIRA Bug 타입, 긴급도 반영
      기능요청 → JIRA Story 타입, 백로그에 추가
      UX개선  → JIRA Task 타입
      중복    → 기존 이슈에 댓글 추가 (새 이슈 안 만듦)

[4] 알림 (Slack)
    → 긴급 버그: 즉시 Slack 알림
    → 기능요청: 주 1회 요약 리포트
    → 감성 분석: 주간 NPS/감성 트렌드
```

### 기술 구현 방안

| 구성요소 | 도구 | 비용 |
|---------|------|------|
| 피드백 수집 | 오픈클로 (Cello) | 무료~유료 |
| AI 분석 | Claude API (Haiku/Sonnet) | 건당 ~1~10원 |
| 자동화 연결 | n8n / Make / Zapier | 무료~월 2만원 |
| 태스크 관리 | JIRA (Atlassian) | 무료 (10명 이하) |
| 알림 | Slack Webhook | 무료 |

### Claude Max 요금제 활용

```
Claude Max 구독 중이라면:

방법 1: Claude API 직접 사용
  → Anthropic API 키 발급 → 자동화 파이프라인에서 호출
  → Max 요금제와 별개로 API 사용량 과금 (별도 비용)

방법 2: Claude Code + MCP 서버 활용
  → JIRA MCP, Slack MCP 연결
  → Claude Code에서 직접 "이 피드백 분석해서 JIRA에 만들어줘" 실행
  → Max 요금제 내 사용량으로 처리 (추가 비용 없음)
  → 단, 수동 트리거 (완전 자동화는 아님)

방법 3: 하이브리드
  → 실시간 자동화: Claude API (Haiku, 저렴) → 자동 분류/생성
  → 주간 리뷰: Claude Code (Max 요금제) → 심층 분석/리포트
```

### 예상 비용 (서비스 런칭 후)

```
월 피드백 500건 기준:

AI 분석 (Claude Haiku): 500건 × ~2원 = ~1,000원
자동화 도구 (n8n):      셀프호스팅 무료 or 클라우드 월 $20
JIRA:                   무료 (10명 이하)
Slack:                  무료
─────────────────────────────────────
합계: ~월 3만원 이내
```

### 구현 우선순위

```
Phase 1 (POC와 동시):
  → 오픈클로 인앱 피드백 위젯 설치
  → Claude API로 피드백 → 분류 → JIRA 생성 파이프라인
  → Slack 긴급 알림

Phase 2 (베타):
  → 중복 감지 (기존 이슈와 벡터 유사도)
  → 주간 감성 분석 리포트 자동 생성
  → 앱스토어 리뷰 자동 수집 연동

Phase 3 (런칭 후):
  → 피드백 기반 자동 우선순위 조정
  → 사용자 투표 기능 → 많이 요청된 기능 자동 상위 배치
```

## 9. 경쟁 분석 & 차별화 전략

### 국내 경쟁사 비교

| 항목 | 재구PICK (지그재그) | 에이블리 | **우리 서비스** |
|------|-------------------|---------|---------------|
| 운영 주체 | 카카오스타일 | 에이블리 | 독립 서비스 |
| 추천 범위 | 지그재그 입점 상품만 | 에이블리 입점 상품만 | **크로스 플랫폼 (무신사+지그재그+W컨셉+29CM 등)** |
| 입력 방식 | 행동 데이터 기반 (구매/클릭) | 행동 데이터 기반 | **사용자가 직접 이미지 업로드** |
| 핵심 데이터 | 구매 이력 수억 건 | 구매 이력 수억 건 | 브랜드/스타일 DB + 이미지 분석 |
| 수익 모델 | 자체 커머스 매출 | 자체 커머스 매출 | 구독 + 어필리에이트 수수료 |
| 강점 | 방대한 구매 데이터 | 방대한 구매 데이터 | **플랫폼 종속 없음** |
| 약점 | 자기 플랫폼만 | 자기 플랫폼만 | 구매 데이터 없음 (초기) |

### 포지셔닝

```
지그재그/에이블리 = 백화점 안의 점원 ("우리 매장에서 이거 어때요?")
우리 서비스       = 네이버 패션 검색 ("이 스타일, 어디서든 찾아줄게")
```

### 차별화 전략: 크로스 플랫폼

```
핵심 가치: "한 곳에 종속되지 않고, 모든 쇼핑몰에서 찾아주는 AI 스타일리스트"

사용자 시나리오:
  1. 인스타에서 마음에 드는 옷 스크린샷
  2. 우리 앱에 업로드
  3. AI가 스타일 분석: "미니멀 캐주얼, 오버핏, 뉴트럴 톤"
  4. 여러 플랫폼에서 유사 상품 추천:
     → 무신사: A 브랜드 셔츠 39,000원
     → W컨셉: B 브랜드 셔츠 52,000원
     → 29CM: C 브랜드 셔츠 45,000원
  5. 가격/브랜드/플랫폼 비교 후 구매 링크 클릭
```

### 핵심 기능 3가지

| 기능 | 설명 | 경쟁사 대비 |
|------|------|------------|
| **이미지 역검색** | "이거 어디 옷이야?" → 사진으로 브랜드/상품 찾기 | 지그재그/에이블리에 없음 |
| **가격대 필터** | "이 스타일을 3만원대로 찾아줘" → 예산 맞춤 추천 | 자체 플랫폼 내에서만 가능 |
| **크로스 플랫폼 비교** | 같은 스타일을 여러 쇼핑몰에서 가격 비교 | 단일 플랫폼 종속 서비스에서 불가 |

### 수익 모델

```
1. 구독료 (주 수입):
   무료:     하루 3건
   베이직:   월 4,900원 (100건)
   프로:     월 9,900원 (500건)

2. 어필리에이트 수수료 (부 수입):
   사용자가 추천 링크로 구매 시 쇼핑몰에서 수수료
   → 쿠팡 파트너스: 3%
   → 무신사 어필리에이트: 5~10%
   → 이중 수익 구조
```

### DB 확보 전략 (단계별)

```
Phase 1 (POC):
  → 50개 인기 브랜드 수동 등록
  → 무신사/지그재그 인기 브랜드 중심

Phase 2 (베타):
  → 어필리에이트 API 연동 (쿠팡 파트너스, 무신사)
  → 상품 데이터 자동 수집 시작

Phase 3 (런칭):
  → 쇼핑몰 제휴 → 공식 API 연동
  → 실시간 재고/가격 반영
```
